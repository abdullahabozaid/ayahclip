## Addendum — 2026-07-20 bug-fix session

Three user-reported bugs were root-caused and fixed directly (blurry preview text → display-density canvas backing; autosave overwriting saved clips → beginNewProject on the verse picker + saveNow identity guard + per-project media-dedup refs + restoreProject defaults; bulk per-clip media reverting → persisted `candidate.styleOverride` applied last on rebuild). One NEW known gap found by the sweep and deliberately deferred:

- **Word-by-word highlight is preview-only**: `wordHighlight` drives a live per-word emphasis in the preview (`StudioPreview.tsx:499-540`) but `renderClipFile` never forwards it and `export.ts` never draws it — the exported MP4 has no traveling highlight, violating preview≠export parity. A correct fix must compute the per-frame word index inside the export frame loop AND include it in the run-length frame signature (or frames collapse and the highlight freezes). Plan it with the same care as plan 005; do not bolt on. Also add `wordHighlight` to the render-cache key when fixed.
